# ロードマップ

## 現状サマリー

企画書のフェーズ区分に照らすと、**Phase 1（MVP）+ Phase 2（ライフイベント）+ Phase 4 の一部（シナリオ比較）**がほぼ完成しています。教育費・住宅ローン・税計算など Phase 5 相当の高度化も一部実装済みです。

---

## 今後取り組むべき内容

### 1. 企画書の未実装機能（優先度：高）

| 項目 | 企画書フェーズ | 現状 | 内容 |
| --- | --- | --- | --- |
| AIチャット相談 | Phase 3 | 未着手 | 企画書のコア差別化要素。ユーザーの状況を前提にしたキャリア・人生設計の対話相談 |
| AI提案のシナリオ反映 | Phase 4 | 未着手 | AIが提案した変更（転職で年収+80万など）をシミュレーション条件に反映 |
| 認証・ユーザー管理 | Phase 1 | 未着手 | 企画書MVP要件だが現在は localStorage のみ。クラウド保存なし |

### 2. シミュレーション精度の向上（優先度：中〜高）

| 項目 | 内容 |
| --- | --- |
| 税制控除の拡充 | 住宅ローン控除、医療費控除、ふるさと納税、生命保険料控除が未対応 |
| 給与所得以外の対応 | 事業所得、譲渡所得、退職所得の計算未対応 |
| 退職金シミュレーション | 退職金の見込み額と税計算 |
| 年金計算の精緻化 | 現在は固定月額のみ。厚生年金・国民年金の概算計算があると良い |
| 月次計算への移行 | 企画書§9-1 で「内部は月次設計が望ましい」と記載。現在は年次計算 |
| 教育費のカスタマイズ | 子ども個別の費用調整が不可。塾・習い事の追加費用も未対応 |

### 3. UI・UX 改善（優先度：中）

| 項目 | 内容 |
| --- | --- |
| 入力離脱の防止 | 企画書§14 リスク1への対策。最低限入力で試算→後から詳細化の段階的 UX |
| ダッシュボードの情報設計 | 比較指標の充実（60歳時点の総資産、支出ピーク年、教育費ピーク年） |
| モバイル対応 | 現状はPC前提の6ステップウィザード。スマホでの入力体験が不十分 |
| サンプルデータのワンクリック読込 | sampleData.ts はあるが UI 上の導線がない |
| 入力バリデーション強化 | フォームの必須チェック・範囲チェックが甘い箇所がある |
| 前提条件の明示 | 企画書§14 リスク2対策。「参考値」であることの表示、計算前提の開示 |
| レポート出力（PDF/印刷） | Phase 5 に記載。結果をまとめて出力する機能 |

### 4. 品質向上（優先度：中）

| 項目 | 内容 |
| --- | --- |
| UIコンポーネントテスト | 現在テストカバレッジはロジック層のみ。React Testing Library によるフォーム・ダッシュボードのテストが皆無 |
| E2Eテスト | ユーザーフロー全体（入力→シミュレーション→シナリオ保存）の通しテストなし |
| ESLint / Prettier 設定 | コードフォーマッターが未設定 |
| アクセシビリティ（a11y） | ARIA属性、キーボード操作、スクリーンリーダー対応が不十分 |
| エラーハンドリング | localStorage 容量超過、データ破損時のリカバリ UI がない |
| パフォーマンス計測 | バンドルサイズ分析、Lighthouse スコアの確認・最適化 |

### 5. インフラ・運用（優先度：低〜中）

| 項目 | 内容 |
| --- | --- |
| HTTPS 対応 | 企画書§10-1 で必須と記載。GitHub Pages は対応済みだが明示的な強制リダイレクトなし |
| バックエンドのDB設計改善 | JSONカラムに全データ保存の非正規化状態。世帯一覧取得もない |
| CI でのテスト実行 | 現在はデプロイのみ。PR時のテスト自動実行がない |
| フロント↔バックエンド連携 | フロントエンドからバックエンド API を呼ぶ UI がない（完全にスタンドアロン） |

## 推奨ロードマップ

### Phase 3（次の重点）

- AIチャット相談機能の実装
- CI でのテスト自動実行追加
- UIコンポーネントテスト拡充

### Phase 4（AI連携強化）

- AI提案 → シナリオ反映
- 比較指標の拡充
- モバイル対応

### Phase 5（高度化）

- 税制控除・年金計算の精緻化
- 認証・クラウド同期
- レポート出力
- E2Eテスト・a11y対応

最大のギャップは AI チャット機能です。企画書のコンセプト「数字とAI対話で見える化する」の後半が未実装であり、差別化のために最優先で取り組むべき領域です。
